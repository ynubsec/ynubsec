## üêõ Reporting Security Issues

### How to Report

**Preferred Method:** Send an encrypted report to **ynubsec@gmail.com**

### What to Include

When reporting a security vulnerability, please provide:

- **Vulnerability Details**: Clear description of the security issue
- **Reproduction Steps**: Detailed steps to reproduce the vulnerability
- **Impact Assessment**: Potential impact and affected components
- **Environment**: Operating system, versions, and configuration
- **Exploitation Difficulty**: Level of expertise required to exploit
- **Proof of Concept**: If available, without active exploitation

### Response Timeline

| Phase | Timeline | Action |
|-------|----------|---------|
| **Acknowledgment** | Within 48 hours | Confirm receipt of your report |
| **Initial Analysis** | 3-5 business days | Preliminary assessment and validation |
| **Coordination** | Ongoing | Discussion of timeline and approach |
| **Resolution** | Varies | Development and testing of fix |
| **Public Disclosure** | Coordinated | Joint announcement with researcher |

## üîç Scope of Security Research

### In-Scope Activities

- Testing your own systems and networks
- Research on publicly available test environments
- Academic security research with proper authorization
- Bug bounty programs and authorized penetration testing
- Security tool development and testing in controlled environments
- Cryptographic analysis of publicly available implementations
- Protocol security analysis with appropriate permissions
- Memory safety research in research environments
- Reverse engineering for security research purposes
- Vulnerability analysis with proper disclosure procedures

### Out-of-Scope Activities

- **Unauthorized Testing**: Any testing of systems without explicit permission
- **Malicious Activity**: Intentional disruption of services or data
- **Privacy Violations**: Accessing or attempting to access private user data
- **Commercial Exploitation**: Using discovered vulnerabilities for financial gain
- **Network Attacks**: DDoS attacks, network scanning without permission
- **Social Engineering**: Attempting to trick users into compromising security
- **Malware Development**: Creating or distributing malicious software
- **Chain Exploitation**: Using one vulnerability to discover unrelated issues
